Export / Import

Migrate secrets and containers between Barbican instances or regions.

{# ── Export ────────────────────────────────────────────────── #}
Export

Download all resources from the current Barbican instance as a JSON file.

Include:
Security notice: The exported file will contain plaintext secret values if payloads are included. Handle the file with the same care as the secrets themselves.
{# ── Import ────────────────────────────────────────────────── #}
Import

Upload a previously exported JSON file to recreate resources in this Barbican instance.

Options:
How it works:
  1. Secrets are created first (matched by name for container references)
  2. Containers are created with references to the newly created secrets
  3. Consumers are re-registered on the new containers
Tip: Use Dry run first to preview the import, then uncheck it and import for real.
{# ── Export format documentation ─────────────────────────────── #}
Export file format 
{
  "version": "1.0",
  "exported_at": "2026-04-25T12:00:00+00:00",
  "source_endpoint": "https://key-manager.regionone.example.com",
  "project_id": "abc123...",
  "project_name": "my-project",
  "secrets": [
    {
      "name": "production/database/password",
      "secret_type": "opaque",
      "algorithm": null,
      "payload": "s3cret-value",
      "payload_content_type": "text/plain"
    }
  ],
  "containers": [
    {
      "name": "tls-bundle",
      "type": "certificate",
      "secret_refs": [
        {"name": "certificate", "secret_name": "production/tls/cert"},
        {"name": "private_key", "secret_name": "production/tls/key"}
      ],
      "consumers": [
        {"name": "load-balancer", "url": "https://lb.example.com"}
      ]
    }
  ]
}
